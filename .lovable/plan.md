

## Plan: PDF Report Download + Visual Alignment with Re-Rooted Aesthetic

Two parts: (1) a downloadable PDF report generated client-side from assessment results, and (2) a visual polish pass across all member app pages to match the marketing website's Re-Rooted aesthetic.

---

### Part 1: Downloadable PDF Report

**New file: `src/utils/assessmentPdf.ts`**
- Uses `jspdf` library (client-side PDF generation, no backend needed)
- Exports a `generateAssessmentPdf(user, assessment, priorityDimensions)` function
- PDF content:
  - Re-Rooted branding header (Deep Blue bar with white text)
  - Title: "Relocation Complexity Score Report"
  - Date generated, user name, country from/to
  - Score prominently displayed with band label (Standard/Enhanced/Intensive/High-Touch)
  - Recommendation text from `getScoreBand()`
  - Interpretation paragraph from `getScoreInterpretation()`
  - Priority Focus Areas list
  - Category-by-category answer breakdown (question text + selected answer label)
  - Footer with "Generated by Re-Rooted®" and date

**Modified: `src/pages/member/AssessmentPage.tsx`**
- Add a "Download PDF Report" button (with Download icon) below the score card in the results view
- Button calls `generateAssessmentPdf()` and triggers browser download

**Install: `jspdf`**

---

### Part 2: Visual Alignment with Re-Rooted Aesthetic

The member app pages currently feel generic compared to the marketing site. Key differences: the marketing site uses bolder typography (font-extrabold/900), more generous spacing, subtle accent colors, and a warmer, more intentional layout. The app pages feel utilitarian.

**Changes across all member pages** (AssessmentPage, MemberHome, ProgressPage, CoachPage, CommunityPage, ProfilePage, MessagesPage, PeerGroupsPage, CulturalCompanion):

1. **Page headers**: Increase title weight to `font-[900]` to match marketing headlines. Add a subtle descriptive subtitle under each page title in `text-muted-foreground`.

2. **Cards**: Remove all remaining `shadow-sm`/`shadow-md` (marketing site is flat). Ensure cards use `bg-card` consistently. Add slightly more padding inside cards (`p-6` instead of `p-4`).

3. **Buttons**: Use `rounded-full` consistently for primary CTAs (already done on Assessment, apply everywhere). Use the brand Deep Blue for primary buttons and Fresh Green for secondary/success actions.

4. **Score display (Assessment results)**: Make the score number larger (`text-6xl`), add the band label as a colored badge (Deep Blue for Standard, Fresh Green for Enhanced, Lavender for Intensive, Warning for High-Touch). Add a subtle divider between score and interpretation.

5. **Progress bars**: Style with brand primary color (already done via CSS variable).

6. **Spacing**: Increase `mb-8` to `mb-10` between major sections for breathing room. Ensure `max-w-2xl` is consistent across all pages.

7. **Empty states**: Use Re-Rooted brand icons/illustrations instead of generic emoji where possible. Use branded language.

8. **Assessment question flow**: Add subtle fade animation between questions using framer-motion. Category badge should use brand accent color.

### Technical details
- `jspdf` is a pure client-side library, no backend function needed
- PDF generation reads from the existing `assessment` object in UserContext
- All visual changes are CSS/Tailwind only, no logic changes
- No database changes required
